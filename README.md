# ei_ts_codegen

A Dart `build_runner` package that automatically converts `json_serializable`-annotated Dart models and enums into fully-decorated TypeScript classes — using `class-transformer`, `class-validator`, and an IST-aware date/GeoPoint utility layer.

Write your models once in Dart. Run one command. Get production-ready TypeScript.

## How It Works

The builder runs in two phases during `dart run build_runner build`:

**Phase 1 — per-file scan (`TsCodegenBuilder`)**
Every `.dart` file is scanned. Classes annotated with `@JsonSerializable` have their field metadata extracted (type, nullability, JSON key overrides, converter annotations, etc.) and written to a sidecar `.ts_meta.json` file in the build cache. All enums in the same library are captured too.

**Phase 2 — aggregate (`TsCodegenAggregateBuilder`)**
All `.ts_meta.json` sidecars are collected across the entire package. Four TypeScript files are then emitted into `lib/ts_output/` (or a configured directory):

```
lib/ts_output/
├── enums.ts    ← all Dart enums as TypeScript string enums
├── utils.ts    ← IST date/GeoPoint transforms and decorator factories
├── models.ts   ← all @JsonSerializable classes as decorated TS classes
└── index.ts    ← re-exports everything from the three files above
```

## Installation

### 1. Add to your Dart package's `pubspec.yaml`

```yaml
dependencies:
  ei_ts_codegen:
    path: ../ei_ts_codegen   # or a pub.dev version once published

dev_dependencies:
  build_runner: ^2.4.9
```

### 2. Add `build.yaml` to the consuming package (optional — only needed to override `output_dir`)

```yaml
# build.yaml in your app package
targets:
  $default:
    builders:
      ei_ts_codegen|ts_codegen_aggregate:
        options:
          output_dir: lib/ts_output   # default — change if needed
```

## Usage

```bash
# One-time generation
dart run build_runner build --delete-conflicting-outputs

# Watch mode during development
dart run build_runner watch --delete-conflicting-outputs
```

The generated `.ts` files will appear in `lib/ts_output/`. Commit them to source control — they are written with `build_to: source`.

## Output Files

### `enums.ts`

Every Dart enum in a scanned library is emitted as a TypeScript string enum so the values are human-readable in JSON payloads.

```typescript
// Auto-generated by ts_codegen. Do not edit manually.

export enum LicenseStatus {
  pending = 'pending',
  approved = 'approved',
  rejected = 'rejected',
}
```

### `utils.ts`

A complete, framework-agnostic utility layer. Emitted verbatim — not derived from your models. Contains:

| Export | Purpose |
|---|---|
| `isIso8601String` | Type guard for ISO 8601 date strings |
| `dateToTimestamp` | Converts raw Firestore-like objects to `Timestamp` |
| `toGeoPoint` | Normalises raw objects to `{ latitude, longitude }` |
| `parseDate` | Parses `Date`, ISO strings, and Firestore Timestamps |
| `formatToIst` | Converts any date value to IST (`Asia/Kolkata`) via `moment-timezone` |
| `TransformDate()` | Decorator — applies `formatToIst` on deserialization |
| `TransformListDate()` | Decorator — applies `formatToIst` to each element of a date array |
| `OptionalValue()` | Decorator — runs validators only when the value is non-null |
| `OptionalNested()` | Decorator — combines `ValidateIf`, `ValidateNested`, and `Type` for nullable nested models |
| `TransformGeoPoint()` | Decorator — converts raw objects to Firestore `GeoPoint` |
| `Default(value)` | Decorator — applies a default value when `exposeDefaultValues` is set |
| `fromFunctionJson` | Deep-converts Timestamps and GeoPoints in raw Cloud Function payloads |
| `toFunctionJson` | Deep-serializes Dates and Timestamps to IST ISO strings for Cloud Functions |

### `models.ts`

One TypeScript class per `@JsonSerializable` Dart class. Each class includes:

- Decorator-annotated fields matching the [decorator decision rules](#decorator-decision-rules)
- A constructor accepting `Partial<ClassName>`
- A `static fromJson(json: unknown): ClassName` method using `plainToInstance`
- A `toJson(): Record<string, any>` method using `instanceToPlain`
- A `static copy(r: ClassName): ClassName` method on any class that extends another model

### `index.ts`

```typescript
// Auto-generated by ts_codegen. Do not edit manually.
export * from './enums';
export * from './utils';
export * from './models';
```

## Type Mapping Reference

| Dart type | TypeScript type |
|---|---|
| `String` | `string` |
| `int`, `double`, `num` | `number` |
| `bool` | `boolean` |
| `dynamic`, `Object` | `unknown` |
| `void` | `void` |
| `Never` | `never` |
| `DateTime` | `Date` |
| `Timestamp` (Firestore) | `Timestamp` |
| `GeoPoint` (Firestore) | `GeoPoint` |
| `List<T>` | `T[]` |
| `Map<K, V>` | `Record<K, V>` |
| `Set<T>` | `Set<T>` |
| `Future<T>` | `Promise<T>` |
| `T?` (nullable) | `T \| undefined` |
| Any other class | TypeScript class of the same name |

Generic type arguments (e.g. the `T` in `List<T?>`) are always treated as non-null — the nullability of the container is what propagates to the TS type suffix.

## Decorator Decision Rules

The emitter chooses decorators deterministically based on the field's inferred type and nullability. In priority order:

### Nested model fields

```
// Required nested model
@ValidateNested()
@Type(() => Address)
address: Address;

// Optional nested model  (Dart: Address?)
@OptionalNested(() => Address)
address?: Address | undefined;

// Required array of nested models  (Dart: List<Address>)
@IsArray()
@ValidateNested({ each: true })
@Type(() => Address)
addresses: Address[];
```

### Date fields

```
// Optional date  (Dart: DateTime?)
@OptionalValue()
@TransformDate()
addedOn?: Date | undefined;

// Required date array  (Dart: List<DateTime>)
@TransformListDate()
dates: Date[];
```

### GeoPoint fields

```
// Optional GeoPoint  (Dart: GeoPoint?)
@OptionalValue()
@TransformGeoPoint()
location?: GeoPoint | undefined;
```

### Enum fields

```
// Optional enum  (Dart: LicenseStatus?)
@OptionalValue()
@IsEnum(LicenseStatus)
licenseStatus?: LicenseStatus | undefined;
```

### Email string fields

Detected when the Dart field carries `@Email` or `@IsEmail`:

```
@OptionalValue()
@IsEmail()
email?: string | undefined;
```

### String array fields

```
@IsArray()
@IsString({ each: true })
deviceTokens: string[];
```

### Primitive fields

```
@IsString()   name: string;
@IsNumber()   pendingAmount: number;
@IsBoolean()  isDeleted: boolean;
```

All optional primitives/enums additionally receive `@OptionalValue()` above the type-specific decorator.

---

## Converter Annotations

If a Dart field carries one of the following converter annotations, the TypeScript type is overridden without any further analysis:

| Dart converter annotation | TypeScript type override |
|---|---|
| `@DateTimeConverter` | `Date` |
| `@DateTimeNullableConverter` | `Date \| undefined` |
| `@TimestampConverter` | `Timestamp` |
| `@GeoPointConverter` | `GeoPoint` |

Converter-annotated date fields still receive `@TransformDate()` so the IST transform is applied on deserialization.

## Configuration

Set `output_dir` in your consuming package's `build.yaml` to change where the TypeScript files are written:

```yaml
targets:
  $default:
    builders:
      ei_ts_codegen|ts_codegen_aggregate:
        options:
          output_dir: lib/generated/ts   # default is lib/ts_output
```

The directory is relative to your package root and must be inside `lib/` (required by `build_to: source`).

## End-to-End Example

### Dart input

```dart
// lib/models/rider.dart

import 'package:json_annotation/json_annotation.dart';

@JsonSerializable()
class EiModel {
  final String id;

  @JsonKey(name: 'added_on')
  final DateTime? addedOn;

  EiModel({required this.id, this.addedOn});
}

@JsonSerializable()
class EiUser extends EiModel {
  @IsEmail()
  final String? email;
  final String? displayName;
  final double pendingAmount;
  final List<String> deviceTokens;
  final bool isDeleted;
  final Address? address;

  EiUser({
    required super.id,
    this.email,
    this.displayName,
    this.pendingAmount = 0.0,
    this.deviceTokens = const [],
    this.isDeleted = false,
    this.address,
  });
}

@JsonSerializable()
class Address {
  final String street;
  final String city;

  Address({required this.street, required this.city});
}
```

```dart
// lib/enums/license_status.dart

enum LicenseStatus { pending, approved, rejected }
```

```typescript
// Auto-generated by ts_codegen. Do not edit manually.

import { Type, instanceToPlain, plainToInstance } from 'class-transformer';
import { IsArray, IsBoolean, IsEmail, IsNumber, IsString, ValidateNested } from 'class-validator';
import { OptionalNested, OptionalValue, TransformDate } from './utils';
import { LicenseStatus } from './enums';

export class EiModel {
  @IsString()
  id: string;

  @OptionalValue()
  @TransformDate()
  addedOn?: Date | undefined;

  constructor(data?: Partial<EiModel>) {
    if (data) {
      Object.assign(this, data);
    }
  }

  static fromJson(json: unknown): EiModel {
    return plainToInstance(EiModel, json, {
      exposeDefaultValues: true,
      enableImplicitConversion: true,
    });
  }

  toJson(): Record<string, any> {
    return instanceToPlain(this, {
      exposeUnsetFields: false,
      enableImplicitConversion: true,
    });
  }
}

export class EiUser extends EiModel {
  @OptionalValue()
  @IsEmail()
  email?: string | undefined;

  @OptionalValue()
  @IsString()
  displayName?: string | undefined;

  @IsNumber()
  pendingAmount: number;

  @IsArray()
  @IsString({ each: true })
  deviceTokens: string[];

  @IsBoolean()
  isDeleted: boolean;

  @OptionalNested(() => Address)
  address?: Address | undefined;

  constructor(data?: Partial<EiUser>) {
    super(data);
    if (data) {
      Object.assign(this, data);
    }
  }

  static fromJson(json: unknown): EiUser { ... }
  toJson(): Record<string, any> { ... }
  static copy(r: EiUser): EiUser { ... }
}
```

## Project Structure

```
ei_ts_codegen/
├── build.yaml                  ← builder registration and phase config
├── pubspec.yaml
└── lib/
    ├── builder.dart            ← entry point: exports tsCodegenBuilder & tsCodegenPostBuilder
    └── src/
        ├── ts_builder.dart     ← Phase 1 (TsCodegenBuilder) and Phase 2 (TsCodegenAggregateBuilder)
        ├── model_emitter.dart  ← metadata extraction + models.ts emission
        ├── enum_emitter.dart   ← metadata extraction + enums.ts emission
        ├── utils_emitter.dart  ← static emit() → verbatim utils.ts content
        └── type_mapper.dart    ← Dart → TS type resolution + FieldMapping analysis
```

## TypeScript Peer Dependencies

The generated code requires these packages in your TypeScript project:

```json
{
  "dependencies": {
    "class-transformer": "^0.5.1",
    "class-validator": "^0.14.0",
    "firebase-admin": "^12.0.0",
    "moment-timezone": "^0.5.45",
    "@google-cloud/bigquery": "^7.0.0",
    "reflect-metadata": "^0.2.0"
  }
}
```

Your TypeScript `tsconfig.json` must also have:

```json
{
  "compilerOptions": {
    "experimentalDecorators": true,
    "emitDecoratorMetadata": true
  }
}
```

And `reflect-metadata` must be imported once at your application entry point:

```typescript
import 'reflect-metadata';
```
